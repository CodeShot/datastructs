\doxysubsection{Detailed Description}
Datastructs uses different nodetypes to implement datastructures.


\begin{DoxyItemize}
\item \mbox{\hyperlink{structsl__node}{sl\+\_\+node}} {\itshape for stack and queue}
\item \mbox{\hyperlink{structdl__node}{dl\+\_\+node}} {\itshape for deque}
\item \mbox{\hyperlink{structdlp__node}{dlp\+\_\+node}} {\itshape for binarytree}
\item \mbox{\hyperlink{structtree__node}{tree\+\_\+node}} {\itshape for tree}
\end{DoxyItemize}

The primary intentended use case of this library would be without interacting with node objects.

At this level only relational aspects such as a datastructure being sequential or hierarchical are fully implemented.

All datastructures have been implemented and tested with explicit pointer types instead of void pointer. Datastructures in pipeline\+:
\begin{DoxyItemize}
\item stack (stable implemented)
\item queue (stable implemented)
\item deque (stable/slow not the nicest implementation, no priority)
\item binarytree (not functional with void pointers)
\item set (depends on binarytree stable when binarytree is stable)
\item bag (N\+OT I\+M\+P\+L\+E\+M\+E\+N\+T\+ED Y\+ET)
\item tree (N\+OT I\+M\+P\+L\+E\+M\+E\+N\+T\+ED Y\+ET)
\end{DoxyItemize}
